Section 4: Figma Auto Layout Audit Skill (New Skill 05)

Dedicated audit for Figma auto layout structure and best practices

05-figma-autolayout-audit.md — 6-Phase Inspection
1
Layout Direction & Wrapping
Is auto layout horizontal or vertical? Does it match the content flow? Are wrapping rules correct for responsive behavior? Flag: horizontal layout used for vertical content stacks, missing wrap on tag/chip groups.
2
Sizing Modes (HUG vs FILL vs FIXED)
CRITICAL Content area set to HUG instead of FILL → footer floats mid-screen
CRITICAL Fixed width on elements that should stretch (buttons, inputs, cards)
MAJOR Parent is FILL but children are all FIXED → wasted space on larger screens
3
Padding & Spacing Consistency
All padding/gaps should follow 8px grid (8, 12, 16, 20, 24, 32, 40, 48). Flag: zero padding on interactive elements, inconsistent gaps between siblings, mixed spacing values within same component type.
4
Alignment & Distribution
Primary alignment (top-left, center, space-between). Counter alignment for cross-axis. Flag: centered content in left-aligned containers, space-between with only 1 child, mismatched alignment across sibling frames.
5
Nesting Depth & Structure
Flag: auto layout nesting deeper than 5 levels (simplify), unnecessary wrapper frames (single child inside auto layout parent), frames without auto layout that should have it.
6
Absolute Positioning Audit
Flag: elements using absolute position that could be in auto layout flow, overlapping elements that break on different screen sizes, absolute-positioned elements without constraints.
