Section 2: Progressive Discovery Flow

How the new one-at-a-time question flow works

Core Questions (Always Asked — One at a Time)
1
Project name & description
What's the project called and what does it do?
2
Platform
Mobile / Web / Cross-platform? (if mobile: iOS, Android, or both?)
3
Target audience
Who are the primary users?
4
Brand colors & visual tone
Primary/secondary colors + tone (Clean, Bold, Dark, Warm, etc.)
5
Navigation pattern
Bottom tabs / Top tabs / Drawer / Sidebar? How many sections?
6
Priority flows/screens
What are the first 2-3 screens or flows to design?
7
Figma icon page name NEW
Which Figma page contains your icons? (e.g., "UI Icons", "Assets/Icons")
Deep Questions (Asked On-Demand When Relevant)
When designing text-heavy screens
→ Brand fonts (primary & secondary)
→ Base text size (14/16/18px)
When setting up design system
→ Corner radius style
→ Spacing density
→ Existing design system status
When building responsive layouts
→ Target screen sizes
→ Figma file structure (single/multi)
When scoping special needs
→ Accessibility requirements
→ Dark mode, i18n, offline
→ User personas
Rule: Deep questions are asked one at a time, only when the current task needs that information. Already-known answers from APP_PLAN.md are skipped.
